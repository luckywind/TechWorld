- ip r /route -n 数据包将发送到哪里？
-  traceroute/tracepath  数据包究竟在哪里丢失？
traceroute google.com
- ss -tulnp 哪些服务正在监听以及监听哪个端口？
- curl 我可以像客户端一样与此服务对话吗？
- dig  /  nslookup DNS 是否在欺骗我？
dig google.com  显示 DNS 服务器、TTL、响应时间。如果主机名解析到错误的 IP 地址或无法解析，则应用程序连接失败。
- tcpdump 显示数据包。无需猜测
tcpdump -i eth0 port 443
