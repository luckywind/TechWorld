# 大页内存 分配/挂载/验证 操作手册

## 一、概念

大页(Hugepage)是 Linux 内核预留的**物理连续、固定大小**的内存页，普通页为 4KB，大页常见 2MB、512MB、1GB。

| 对比 | 普通页 4KB | 大页 2MB | 大页 512MB |
|------|-----------|---------|-----------|
| 1GB 需多少页 | 262,144 个 | 512 个 | 2 个 |
| TLB 条目占用 | 极多 | 少 | 极少 |
| 物理连续性 | 不保证 | 保证连续 | 保证连续 |
| 内存浪费风险 | 低 | 中 | 高（未用满即浪费） |
| 适用场景 | 通用 | x86 通用 | ARM64 服务器 |

## 二、查看

### 查看当前大页状态

```bash
cat /proc/meminfo | grep -i huge
```

关键字段：

| 字段 | 含义 |
|------|------|
| `Hugepagesize` | 系统**默认**大页大小(kB) |
| `HugePages_Total` | 已分配大页总数 |
| `HugePages_Free` | 空闲大页数（0 则无法使用） |
| `Hugetlb` | 大页总内存量(kB) |

输出示例：

```
HugePages_Total:    1024      ← 1024 个大页
HugePages_Free:     1024      ← 全部空闲
Hugepagesize:       2048 kB   ← 每个 2MB
Hugetlb:         2097152 kB   ← 总计 2GB
```

### 查看系统支持哪些大页大小

```bash
ls /sys/kernel/mm/hugepages/
# hugepages-2048kB    ← 支持 2MB
# hugepages-524288kB  ← 支持 512MB（ARM64常见）
# hugepages-1048576kB ← 支持 1GB
```

### 查看每个 NUMA node 的大页分配

```bash
ls /sys/devices/system/node/node*/hugepages/
```

### 查看挂载状态

```bash
mount | grep hugetlbfs
# hugetlbfs on /dev/hugepages type hugetlbfs (rw,relatime,pagesize=2M)
```

关注 `pagesize=` 的值，必须和分配的大页大小一致。

## 三、分配

### 全局分配（所有 NUMA node 共用）

```bash
# 分配 1024 个 2MB 大页 = 2GB
echo 1024 > /sys/kernel/mm/hugepages/hugepages-2048kB/nr_hugepages

# 分配 4 个 512MB 大页 = 2GB
echo 4 > /sys/kernel/mm/hugepages/hugepages-524288kB/nr_hugepages
```

### 按 NUMA node 分配

```bash
# 给 node0 分配 512 个 2MB 大页(1GB)
echo 512 > /sys/devices/system/node/node0/hugepages/hugepages-2048kB/nr_hugepages

# 给 node1 分配 512 个 2MB 大页(1GB)
echo 512 > /sys/devices/system/node/node1/hugepages/hugepages-2048kB/nr_hugepages
```

> **建议**：按 NUMA node 分配，让每个 node 的 CPU 访问本地大页，避免跨 node 访存降低性能。

### 清除大页

```bash
# 清空 2MB 大页
echo 0 > /sys/kernel/mm/hugepages/hugepages-2048kB/nr_hugepages
```

> 如有进程正在使用大页（DPDK 应用运行中），**不能清除**，会报错。

### 持久化（重启不丢失）

在 `/etc/sysctl.conf` 或 `/etc/sysctl.d/99-hugepages.conf` 中写入：

```bash
# 分配 1024 个 2MB 大页
vm.nr_hugepages = 1024

# 按 NUMA node 分配
vm.nr_hugepages_node0 = 512
vm.nr_hugepages_node1 = 512
```

或通过 grub 内核参数（更早生效）：

```bash
# /etc/default/grub
GRUB_CMDLINE_LINUX="... default_hugepagesz=2M hugepagesz=2M hugepages=1024"
```

## 四、挂载

### 挂载 hugetlbfs

DPDK 默认使用 `/dev/hugepages`：

```bash
mkdir -p /dev/hugepages
mount -t hugetlbfs -o pagesize=2M nodev /dev/hugepages
```

| 参数 | 说明 |
|------|------|
| `-t hugetlbfs` | 文件系统类型 |
| `-o pagesize=2M` | **必须和大页分配的大小一致** |
| `nodev` | 不关联具体设备 |
| `/dev/hugepages` | 挂载路径（DPDK 默认） |

### 常见错误：pagesize 和分配大小不一致

```bash
# 分配了 512MB 大页，但挂载指定了 2M → DPDK 报错
mount -t hugetlbfs -o pagesize=2M nodev /dev/hugepages

# 正确做法：pagesize 和分配的大小匹配
mount -t hugetlbfs -o pagesize=512M nodev /dev/hugepages
```

### 多页大小同时挂载

如果系统同时用了 2MB 和 512MB 大页，分别挂载到不同目录：

```bash
mkdir -p /dev/hugepages-2M  /dev/hugepages-512M
mount -t hugetlbfs -o pagesize=2M   nodev /dev/hugepages-2M
mount -t hugetlbfs -o pagesize=512M nodev /dev/hugepages-512M
```

### 持久化（重启不丢失）

`/etc/fstab` 添加：

```
nodev /dev/hugepages hugetlbfs pagesize=2M 0 0
```

### 卸载

```bash
umount /dev/hugepages
```

> 有进程在使用 hugetlbfs 时（DPDK 正在跑）**不能卸载**，先停进程。

## 五、自动化脚本

```bash
#!/bin/bash
# 一键配置大页内存

HP_SIZE=${1:-2048}     # 大页大小(kB)，默认 2MB=2048，512MB=524288
HP_COUNT=${2:-1024}    # 大页数量
HP_MOUNT=${3:-/dev/hugepages}

# 1. 分配
echo "$HP_COUNT" > /sys/kernel/mm/hugepages/hugepages-${HP_SIZE}kB/nr_hugepages 2>/dev/null || {
    echo "错误: 大页大小 ${HP_SIZE}kB 不支持"
    echo "支持的大小: $(ls /sys/kernel/mm/hugepages/)"
    exit 1
}

# 2. 挂载
mkdir -p "$HP_MOUNT"
PS_MB=$((HP_SIZE / 1024))
mount | grep -q "$HP_MOUNT" || mount -t hugetlbfs -o pagesize=${PS_MB}M nodev "$HP_MOUNT"

# 3. 验证
echo "=== 大页状态 ==="
cat /proc/meminfo | grep HugePages
echo "=== 挂载状态 ==="
mount | grep hugetlbfs
```

## 六、快速验证

```bash
# 三步确认大页就绪
cat /proc/meminfo | grep HugePages_Free   # >0 表示有大页可用
mount | grep hugetlbfs                    # 必须有输出
cat /proc/meminfo | grep Hugepagesize     # 记下大小，和挂载的 pagesize= 对比
```

三项都就绪，DPDK 才能正常启动。
