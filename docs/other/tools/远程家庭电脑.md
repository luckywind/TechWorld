# 部署

[参考](https://blog.stackademic.com/make-your-home-computer-reachable-from-anywhere-no-public-ip-no-router-touching-3c1295c8811b)、

双端都需要安装

- **Tailscale** ：使您的笔记本电脑和家用电脑表现得如同在同一个私有局域网中一样（即使跨越互联网，跨越具有定期更改 IP 地址的 CGNAT，跨越您的设备，只要每个设备都安装并运行了 Tailscale）。

- **NoMachine** ：提供稳定的远程桌面会话（打字感觉正常，剪贴板工作正常，OAuth/浏览器功能运行正常，复制/粘贴功能开箱即用，无需配置）。

两端登录同一个账号

两台 Mac 处于不同的局域网（NAT）后，就像住在两个没有互访权限的小区，NoMachine 默认的“自动发现”功能失效是必然的。要让它们跨网络通信，核心思路是使用tailscale**建立一条“虚拟专线”**，让两台电脑以为彼此在同一个局域网内。

原理： Tailscale 会自动尝试建立 P2P 直连（UDP 打洞，打洞需要 UDP 41641 端口，确保放行），如果失败会自动通过 DERP 中继服务器中转，你无需关心网络细节。

### windows
- 安装 tailscale

```shell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser 
iwr -useb get.scoop.sh | iex 
scoop bucket add extras 
scoop install tailscale
```

获取 ip 地址100.xxx.xx.xxx,你的 mac 应该可以直接访问这个 IP 地址

`tailscale ip -4`  

`tailscale status`会列出所有机器

- 安装 nomachine

会在100.xxx.xx.xxx 上暴漏 4000 端口

### mac

-  安装 tailscale

```shell
brew install --cask tailscale-app 
open -a Tailscale
```

- 安装 nomachine

`brew install --cask nomachine`

测试 nomachine 的默认端口是否连通

`nc -vz 100.xxx.xx.xxx 4000`

# 使用

tailscale status  判断 tailscale 状态

 nc -vz/vuz <tailscale-ip> 4000  判断远端 nomachine连接状态(判断 tcp 或者 udp)

netstat -an | grep 4000 判断本机端口监听状态

# 问题

## 慢

tailscale ping 100.69.213.1
pong from macbook-pro-1 (100.69.213.1) via DERP(sfo) in 543ms
pong from macbook-pro-1 (100.69.213.1) via DERP(sfo) in 536ms
pong from macbook-pro-1 (100.69.213.1) via DERP(sfo) in 1.14s
pong from macbook-pro-1 (100.69.213.1) via DERP(sfo) in 433ms
pong from macbook-pro-1 (100.69.213.1) via DERP(sfo) in 422ms
pong from macbook-pro-1 (100.69.213.1) via DERP(sfo) in 451ms
pong from macbook-pro-1 (100.69.213.1) via DERP(sfo) in 428ms
pong from macbook-pro-1 (100.69.213.1) via DERP(sfo) in 390ms

这意味着两台 Mac 之间的连接**没有建立 P2P 直连**，而是绕道到了 Tailscale 位于美国旧金山（SFO）的 DERP 中继服务器。

当 Tailscale 无法在两台设备间直接打通 UDP 连接（即“打洞”失败）时，它会自动回退到通过全球的 DERP 服务器进行 TCP 中继

常见原因有：

1. **对称型 NAT**：你家里或办公室的路由器使用了最严格的 NAT 类型，阻止了 P2P 连接。
2. **防火墙阻止了关键端口**：Tailscale 用于打洞的 UDP 41641 等端口被防火墙拦截。
3. **运营商限制**：部分运营商（尤其是蜂窝网络或企业网络）会限制或干扰 P2P UDP 流量。



解决方法：

1. 使用 TCP
   编辑连接->Configuration->Multimedia 取消勾选 UDP

2. 调节画质

   Ctrl+Alt+0呼出菜单后：

   1. 找到 **Display**（显示） -> **Display settings**（显示设置）。
   2. 在这里你会看到一个 **Quality**（质量）滑块。
   3. **向左拖动滑块**，即可降低画质，提升高延迟下的流畅度。
